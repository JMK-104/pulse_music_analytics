# 7.1 Fact Table — `fact_listening`

## Business Purpose

The `fact_listening` table is the central fact table of the Pulse Music Analytics warehouse.

Each record represents a single song playback event and captures the measurable aspects of user listening behavior. It serves as the primary source for reporting on music engagement, content performance, user activity, and platform usage.

This table is expected to be the largest table in the warehouse.

---

## Grain

**One row represents one song playback event.**

Example:

A user presses play on a song.

↓

One record is written to `fact_listening`.

---

## Source

Derived primarily from the operational `Listening_History` table during ETL, with surrogate key lookups performed against the Gold Layer dimensions.

---

## Primary Key

`listening_key`

---

## Foreign Keys

| Foreign Key  | References              |
| ------------ | ----------------------- |
| date_key     | dim_date                |
| time_key     | dim_time                |
| user_key     | dim_user                |
| song_key     | dim_song                |
| device_key   | dim_device              |
| playlist_key | dim_playlist (nullable) |
| plan_key     | dim_plan                |
| source_key   | dim_source              |

---

## Measures

| Measure        | Description                                                      |
| -------------- | ---------------------------------------------------------------- |
| seconds_played | Number of seconds listened before playback ended or stopped.     |
| completed_flag | Indicates whether the listener reached the completion threshold. |
| skipped_flag   | Indicates whether the song was skipped before completion.        |

---

## Column Definitions

| Column         | Data Type | Nullable |  PK |  FK | Default        | Description                                                         |
| -------------- | --------- | :------: | :-: | :-: | -------------- | ------------------------------------------------------------------- |
| listening_key  | BIGINT    |    No    |  ✓  |     | Auto-generated | Surrogate key uniquely identifying each playback event.             |
| date_key       | INTEGER   |    No    |     |  ✓  |                | References `dim_date`.                                              |
| time_key       | INTEGER   |    No    |     |  ✓  |                | References `dim_time`.                                              |
| user_key       | INTEGER   |    No    |     |  ✓  |                | References `dim_user`.                                              |
| song_key       | INTEGER   |    No    |     |  ✓  |                | References `dim_song`.                                              |
| device_key     | INTEGER   |    No    |     |  ✓  |                | References `dim_device`.                                            |
| playlist_key   | INTEGER   |    Yes   |     |  ✓  | NULL           | References `dim_playlist` when playback originated from a playlist. |
| plan_key       | INTEGER   |    No    |     |  ✓  |                | References `dim_plan`.                                              |

---

## Business Rules

* One row equals one playback event.
* Every playback must reference valid dimension records.
* Playlist is optional because not all playback originates from a playlist.
* `seconds_played` cannot exceed the song duration.
* A completed song cannot be marked as skipped.
* Songs played from Search, Album, Radio, Recommendation, or Library have a NULL playlist.

---

## Data Quality Rules

| Rule                                      | Expected Result |
| ----------------------------------------- | --------------- |
| No duplicate listening_key values         | Pass            |
| All foreign keys exist                    | Pass            |
| Seconds played > 0                        | Pass            |
| Seconds played ≤ song duration            | Pass            |
| Completed and skipped cannot both be TRUE | Pass            |

---

## Estimated Size

Approximately **15–20 million rows**.
