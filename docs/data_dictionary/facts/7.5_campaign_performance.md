# 7.5 Fact Table — `fact_campaign_performance`

## Business Purpose

The `fact_campaign_performance` table stores daily performance metrics for marketing campaigns.

It supports analysis of campaign effectiveness, customer acquisition, advertising spend, and return on investment (ROI) across marketing channels.

Unlike transactional fact tables, this table summarizes campaign activity for a given reporting period.

---

## Grain

**One row represents one campaign on one calendar day.**

Example:

Campaign: Summer Premium Promotion

Date: 2025-06-15

↓

One summary record is created containing that day's performance metrics.

---

## Source

Derived from marketing analytics systems during ETL.

---

## Primary Key

`campaign_performance_key`

---

## Foreign Keys

| Foreign Key  | References   |
| ------------ | ------------ |
| date_key     | dim_date     |
| campaign_key | dim_campaign |

---

## Measures

| Measure           | Description                                 |
| ----------------- | ------------------------------------------- |
| impressions       | Number of advertisement impressions served. |
| clicks            | Number of advertisement clicks.             |
| conversions       | Number of successful user conversions.      |
| ad_spend          | Advertising spend for the reporting period. |
| revenue_generated | Revenue attributed to the campaign.         |

---

## Column Definitions

| Column                   | Data Type     | Nullable |  PK |  FK | Default        | Description                                |
| ------------------------ | ------------- | :------: | :-: | :-: | -------------- | ------------------------------------------ |
| campaign_performance_key | BIGINT        |    No    |  ✓  |     | Auto-generated | Surrogate key.                             |
| date_key                 | INTEGER       |    No    |     |  ✓  |                | References `dim_date`.                     |
| campaign_key             | INTEGER       |    No    |     |  ✓  |                | References `dim_campaign`.                 |
| impressions              | INTEGER       |    No    |     |     | 0              | Number of impressions.                     |
| clicks                   | INTEGER       |    No    |     |     | 0              | Number of clicks.                          |
| conversions              | INTEGER       |    No    |     |     | 0              | Number of successful conversions.          |
| ad_spend                 | NUMERIC(12,2) |    No    |     |     |                | Advertising cost for the reporting period. |
| revenue_generated        | NUMERIC(12,2) |    No    |     |     |                | Revenue attributed to the campaign.        |

---

## Business Rules

* One row represents one campaign for one reporting day.
* Clicks cannot exceed impressions.
* Conversions cannot exceed clicks.
* Advertising spend cannot be negative.
* Revenue generated may be zero.

---

## Data Quality Rules

| Rule                                         | Expected Result |
| -------------------------------------------- | --------------- |
| No duplicate campaign_performance_key values | Pass            |
| Clicks ≤ impressions                         | Pass            |
| Conversions ≤ clicks                         | Pass            |
| Ad spend ≥ 0                                 | Pass            |
| Revenue generated ≥ 0                        | Pass            |

---

## Estimated Size

Approximately **25,000 rows**.
