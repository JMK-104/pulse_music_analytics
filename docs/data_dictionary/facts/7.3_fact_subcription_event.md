# 7.3 Fact Table — `fact_subscription_event`

## Business Purpose

The `fact_subscription_event` table records the lifecycle of user subscriptions.

It supports reporting on subscriptions, upgrades, downgrades, renewals, cancellations, churn, and customer retention.

---

## Grain

**One row represents one subscription lifecycle event.**

Example:

User upgrades from Free to Premium.

↓

One event is recorded.

---

## Source

Derived from the operational `User_Subscriptions` table during ETL.

---

## Primary Key

`subscription_event_key`

---

## Foreign Keys

| Foreign Key | References |
| ----------- | ---------- |
| date_key    | dim_date   |
| user_key    | dim_user   |
| plan_key    | dim_plan   |

---

## Measures

This fact table contains no additive numeric measures.

Its primary analytical value comes from counting events by type.

---

## Column Definitions

| Column                 | Data Type   | Nullable |  PK |  FK | Default        | Description                                     |
| ---------------------- | ----------- | :------: | :-: | :-: | -------------- | ----------------------------------------------- |
| subscription_event_key | BIGINT      |    No    |  ✓  |     | Auto-generated | Surrogate key.                                  |
| date_key               | INTEGER     |    No    |     |  ✓  |                | References `dim_date`.                          |
| user_key               | INTEGER     |    No    |     |  ✓  |                | References `dim_user`.                          |
| plan_key               | INTEGER     |    No    |     |  ✓  |                | References `dim_plan`.                          |
| event_type             | VARCHAR(30) |    No    |     |     |                | Subscription event type.                        |
| auto_renew             | BOOLEAN     |    No    |     |     | TRUE           | Indicates whether automatic renewal is enabled. |

---

## Business Rules

Supported event types:

* Started
* Renewed
* Upgraded
* Downgraded
* Cancelled

Additional rules:

* Event type must use approved values.
* Every event belongs to exactly one user.
* Every event references one subscription plan.

---
