# 7.6 Fact Table — `fact_podcast_play`

## Business Purpose

The `fact_podcast_play` table records individual podcast listening events on the Pulse Music platform.

It enables podcast engagement analysis, completion rates, listening behavior, device usage, and subscription plan comparisons for podcast content.

Although structurally similar to `fact_listening`, it models podcast consumption independently to support content-specific reporting and future expansion.

---

## Grain

**One row represents one podcast episode playback event.**

Example:

A user begins playing a podcast episode.

↓

One record is written to `fact_podcast_play`.

---

## Source

Derived from the operational `Podcast_Listening_History` table during ETL.

---

## Primary Key

`podcast_play_key`

---

## Foreign Keys

| Foreign Key | References  |
| ----------- | ----------- |
| date_key    | dim_date    |
| time_key    | dim_time    |
| user_key    | dim_user    |
| episode_key | dim_episode |
| device_key  | dim_device  |
| plan_key    | dim_plan    |

---

## Measures

| Measure        | Description                                         |
| -------------- | --------------------------------------------------- |
| seconds_played | Number of seconds listened.                         |
| completed_flag | Indicates whether the episode was completed.        |
| skipped_flag   | Indicates whether playback ended before completion. |

---

## Column Definitions

| Column           | Data Type | Nullable |  PK |  FK | Default        | Description                                            |
| ---------------- | --------- | :------: | :-: | :-: | -------------- | ------------------------------------------------------ |
| podcast_play_key | BIGINT    |    No    |  ✓  |     | Auto-generated | Surrogate key uniquely identifying the playback event. |
| date_key         | INTEGER   |    No    |     |  ✓  |                | References `dim_date`.                                 |
| time_key         | INTEGER   |    No    |     |  ✓  |                | References `dim_time`.                                 |
| user_key         | INTEGER   |    No    |     |  ✓  |                | References `dim_user`.                                 |
| episode_key      | INTEGER   |    No    |     |  ✓  |                | References `dim_episode`.                              |
| device_key       | INTEGER   |    No    |     |  ✓  |                | References `dim_device`.                               |
| plan_key         | INTEGER   |    No    |     |  ✓  |                | References `dim_plan`.                                 |
| seconds_played   | INTEGER   |    No    |     |     |                | Number of seconds listened.                            |
| completed_flag   | BOOLEAN   |    No    |     |     | FALSE          | TRUE if the completion threshold was reached.          |
| skipped_flag     | BOOLEAN   |    No    |     |     | FALSE          | TRUE if playback ended before completion.              |

---

## Business Rules

* One row represents one podcast playback event.
* Every playback must reference a valid episode.
* Playback duration cannot exceed the episode duration.
* A completed episode cannot be marked as skipped.
* Every playback must belong to one user.

---

## Data Quality Rules

| Rule                                                 | Expected Result |
| ---------------------------------------------------- | --------------- |
| No duplicate podcast_play_key values                 | Pass            |
| Playback duration > 0                                | Pass            |
| Playback duration ≤ episode duration                 | Pass            |
| All foreign keys exist                               | Pass            |
| Completed and skipped flags are logically consistent | Pass            |

---

## Estimated Size

Approximately **2–4 million rows**.
