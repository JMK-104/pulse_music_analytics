# 7.2 Fact Table — `fact_session`

## Business Purpose

The `fact_session` table stores summary information for each user application session.

It enables analysis of user engagement, session frequency, listening intensity, and application usage patterns.

---

## Grain

**One row represents one user application session.**

Example:

User opens the app.

↓

Listens for 42 minutes.

↓

Closes the app.

↓

One record is created.

---

## Source

Derived from the operational `Sessions` table during ETL.

---

## Primary Key

`session_key`

---

## Foreign Keys

| Foreign Key | References |
| ----------- | ---------- |
| date_key    | dim_date   |
| time_key    | dim_time   |
| user_key    | dim_user   |
| device_key  | dim_device |

---

## Measures

| Measure                  | Description                                     |
| ------------------------ | ----------------------------------------------- |
| session_duration_seconds | Total session duration.                         |
| songs_played             | Number of songs played.                         |
| podcasts_played          | Number of podcast episodes played.              |
| searches_performed       | Number of searches executed during the session. |

---

## Column Definitions

| Column                   | Data Type | Nullable |  PK |  FK | Default        | Description                        |
| ------------------------ | --------- | :------: | :-: | :-: | -------------- | ---------------------------------- |
| session_key              | BIGINT    |    No    |  ✓  |     | Auto-generated | Surrogate key.                     |
| date_key                 | INTEGER   |    No    |     |  ✓  |                | References `dim_date`.             |
| time_key                 | INTEGER   |    No    |     |  ✓  |                | References `dim_time`.             |
| user_key                 | INTEGER   |    No    |     |  ✓  |                | References `dim_user`.             |
| device_key               | INTEGER   |    No    |     |  ✓  |                | References `dim_device`.           |
| session_duration_seconds | INTEGER   |    No    |     |     |                | Total session duration.            |
| songs_played             | INTEGER   |    No    |     |     | 0              | Number of songs played.            |
| podcasts_played          | INTEGER   |    No    |     |     | 0              | Number of podcast episodes played. |
| searches_performed       | INTEGER   |    No    |     |     | 0              | Number of searches performed.      |

---

## Business Rules

* One row equals one application session.
* Session duration must be greater than zero.
* Sessions may contain music only, podcasts only, or both.
* Every session must belong to exactly one user.

---

## Data Quality Rules

| Rule                            | Expected Result |
| ------------------------------- | --------------- |
| No duplicate session_key values | Pass            |
| Session duration > 0            | Pass            |
| All foreign keys exist          | Pass            |
| Counts cannot be negative       | Pass            |

---

## Estimated Size

Approximately **3–5 million rows**.
