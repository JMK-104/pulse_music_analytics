# 6.11 Dimension Table — `dim_playlist`

## Business Purpose

The `dim_playlist` table stores descriptive information about playlists available on the Pulse Music platform.

It supports analysis of playlist engagement, discovery behavior, and the impact of algorithmic versus user-created playlists.

---

## Grain

**One row represents one playlist.**

Example:

| playlist_key | playlist_name   |
| ------------ | --------------- |
| 1            | Discover Weekly |
| 2            | Workout Hits    |
| 3            | User Favorites  |

---

## Source

Derived from the operational `Playlists` table during ETL.

---

## Primary Key

`playlist_key`

---

## Referenced By

* fact_listening

---

## Column Definitions

| Column        | Data Type    | Nullable |  PK |  FK | Default        | Description                                            |
| ------------- | ------------ | :------: | :-: | :-: | -------------- | ------------------------------------------------------ |
| playlist_key  | INTEGER      |    No    |  ✓  |     | Auto-generated | Surrogate key used throughout the warehouse.           |
| playlist_id   | VARCHAR(20)  |    No    |     |     |                | Business identifier from the operational system.       |
| playlist_name | VARCHAR(255) |    No    |     |     |                | Playlist title.                                        |
| creator_type  | VARCHAR(50)  |    No    |     |     |                | Type of playlist creator.                              |
| public        | BOOLEAN      |    No    |     |     | TRUE           | Indicates whether the playlist is publicly accessible. |
| creation_date | DATE         |    No    |     |     |                | Date the playlist was created.                         |

---

## Business Rules

Supported creator types:

* User
* Editorial
* Algorithmic

Additional rules:

* Playlist IDs must be unique.
* Public playlists may be surfaced in recommendations.
* Algorithmic playlists do not have a human creator.

---
