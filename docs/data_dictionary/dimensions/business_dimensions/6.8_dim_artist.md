# 6.8 Dimension Table — `dim_artist`

## Business Purpose

The `dim_artist` table stores descriptive information about music artists available on the Pulse Music platform.

It supports analysis of content performance, artist popularity, listener engagement, and geographic trends.

---

## Grain

**One row represents one artist.**

Example:

| artist_key | artist_name     |
| ---------- | --------------- |
| 1          | The Midnight    |
| 2          | Aurora          |
| 3          | Imagine Dragons |

---

## Source

Derived from the operational `Artists` table during ETL.

---

## Primary Key

`artist_key`

---

## Referenced By

* dim_song

---

## Column Definitions

| Column      | Data Type    | Nullable |  PK |  FK | Default        | Description                                      |
| ----------- | ------------ | :------: | :-: | :-: | -------------- | ------------------------------------------------ |
| artist_key  | INTEGER      |    No    |  ✓  |     | Auto-generated | Surrogate key used throughout the warehouse.     |
| artist_id   | VARCHAR(20)  |    No    |     |     |                | Business identifier from the operational system. |
| artist_name | VARCHAR(255) |    No    |     |     |                | Official artist name.                            |
| country     | VARCHAR(100) |    Yes   |     |     |                | Artist's country of origin.                      |
| label       | VARCHAR(255) |    Yes   |     |     |                | Record label associated with the artist.         |
| debut_year  | SMALLINT     |    Yes   |     |     |                | Year the artist began their professional career. |

---

## Business Rules

* Each artist must have a unique `artist_id`.
* Artist names must be standardized during ETL.
* Unknown artist values should be assigned to an "Unknown Artist" record rather than stored as NULL in downstream fact relationships.
* Debut year cannot exceed the current year.

---

## Data Quality Rules

| Rule                                    | Expected Result |
| --------------------------------------- | --------------- |
| No duplicate artist_id values           | Pass            |
| Artist name cannot be NULL              | Pass            |
| Debut year is valid                     | Pass            |
| Artist country uses standardized values | Pass            |

---

Approximately **600 rows**.
