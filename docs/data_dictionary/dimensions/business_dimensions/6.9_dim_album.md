# 6.9 Dimension Table — `dim_album`

## Business Purpose

The `dim_album` table stores descriptive information about music albums available on the Pulse Music platform.

It supports album-level analysis including listening performance, release trends, and comparisons between album formats.

---

## Grain

**One row represents one album.**

Example:

| album_key | album_name             |
| --------- | ---------------------- |
| 1         | Random Access Memories |
| 2         | Future Nostalgia       |
| 3         | Discovery              |

---

## Source

Derived from the operational `Albums` table during ETL.

---

## Primary Key

`album_key`

---

## Referenced By

* dim_song

---

## Column Definitions

| Column       | Data Type    | Nullable |  PK |  FK | Default        | Description                                      |
| ------------ | ------------ | :------: | :-: | :-: | -------------- | ------------------------------------------------ |
| album_key    | INTEGER      |    No    |  ✓  |     | Auto-generated | Surrogate key used throughout the warehouse.     |
| album_id     | VARCHAR(20)  |    No    |     |     |                | Business identifier from the operational system. |
| album_name   | VARCHAR(255) |    No    |     |     |                | Official album title.                            |
| album_type   | VARCHAR(50)  |    No    |     |     |                | Album format classification (LP, EP, Single).    |
| release_date | DATE         |    No    |     |     |                | Date the album was released.                     |

---

## Business Rules

* Each album must have a unique `album_id`.
* Album type must be one of the supported categories:

  * LP
  * EP
  * Single
* Release date cannot occur in the future.
* An album may contain multiple songs.

---

## Data Quality Rules

| Rule                               | Expected Result |
| ---------------------------------- | --------------- |
| No duplicate album_id values       | Pass            |
| Album name cannot be NULL          | Pass            |
| Release date is valid              | Pass            |
| Album type matches approved values | Pass            |

---

## Estimated Size

Approximately **1,200 rows**.
