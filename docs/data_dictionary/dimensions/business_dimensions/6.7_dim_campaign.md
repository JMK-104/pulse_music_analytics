# 6.7 Dimension Table — `dim_campaign`

## Business Purpose

The `dim_campaign` table stores descriptive information about marketing campaigns used to acquire and engage Pulse Music users.

This dimension enables analysis of marketing performance across different channels, campaign types, objectives, and target markets when combined with `fact_campaign_performance`.

---

## Grain

**One row represents one marketing campaign.**

Example:

| campaign_key | campaign_name                |
| ------------ | ---------------------------- |
| 1            | Summer Premium Promotion     |
| 2            | Student Acquisition Campaign |
| 3            | Podcast Awareness Campaign   |

---

## Source

Derived from the operational `Marketing_Campaigns` table during ETL.

---

## Primary Key

`campaign_key`

---

## Referenced By

* fact_campaign_performance

---

## Column Definitions

| Column         | Data Type    | Nullable |  PK |  FK | Default        | Description                                                           |
| -------------- | ------------ | :------: | :-: | :-: | -------------- | --------------------------------------------------------------------- |
| campaign_key   | INTEGER      |    No    |  ✓  |     | Auto-generated | Surrogate key used throughout the warehouse.                          |
| campaign_id    | VARCHAR(20)  |    No    |     |     |                | Business identifier from the operational system.                      |
| campaign_name  | VARCHAR(255) |    No    |     |     |                | Name of the marketing campaign.                                       |
| channel        | VARCHAR(50)  |    No    |     |     |                | Marketing channel used (e.g., Google Ads, TikTok, Email).             |
| campaign_type  | VARCHAR(50)  |    No    |     |     |                | Classification of campaign (Acquisition, Retention, Brand Awareness). |
| objective      | VARCHAR(100) |    No    |     |     |                | Primary goal of the campaign.                                         |
| target_country | VARCHAR(100) |    Yes   |     |     |                | Geographic audience targeted by the campaign.                         |

---

## Business Rules

* Each campaign must have a unique `campaign_id`.
* Campaign channels must be standardized during ETL.
* Campaign type must belong to an approved list of categories.
* A campaign may target one or more countries in future implementations. The current model supports one primary target country.

---

## Data Quality Rules

| Rule                                   | Expected Result |
| -------------------------------------- | --------------- |
| No duplicate campaign_id values        | Pass            |
| Campaign name cannot be NULL           | Pass            |
| Channel must use approved values       | Pass            |
| Campaign type must use approved values | Pass            |

---

## Estimated Size

Approximately **70 rows**.
