# 6.10 Dimension Table — `dim_song`

## Business Purpose

The `dim_song` table stores descriptive information about individual songs available on the Pulse Music platform.

It serves as the primary content dimension for music listening analysis and allows analysts to evaluate performance by song attributes such as genre, release period, explicit content status, artist, and album.

This dimension links music content to both `dim_artist` and `dim_album`.

---

## Grain

**One row represents one unique song.**

Example:

| song_key | song_title      |
| -------- | --------------- |
| 1        | Blinding Lights |
| 2        | Levitating      |
| 3        | Midnight City   |

---

## Source

Derived from the operational `Songs` table during ETL.

---

## Primary Key

`song_key`

---

## Referenced By

* fact_listening

---

## References

* `artist_key` → dim_artist
* `album_key` → dim_album

---

## Column Definitions

| Column           | Data Type    | Nullable |  PK |  FK | Default        | Description                                           |
| ---------------- | ------------ | :------: | :-: | :-: | -------------- | ----------------------------------------------------- |
| song_key         | INTEGER      |    No    |  ✓  |     | Auto-generated | Surrogate key used throughout the warehouse.          |
| song_id          | VARCHAR(20)  |    No    |     |     |                | Business identifier from the operational system.      |
| artist_key       | INTEGER      |    No    |     |  ✓  |                | Foreign key referencing the artist dimension.         |
| album_key        | INTEGER      |    Yes   |     |  ✓  |                | Foreign key referencing the album dimension.          |
| song_title       | VARCHAR(255) |    No    |     |     |                | Official song title.                                  |
| genre            | VARCHAR(50)  |    No    |     |     |                | Primary music genre classification.                   |
| duration_seconds | INTEGER      |    No    |     |     |                | Total length of the song in seconds.                  |
| release_year     | SMALLINT     |    No    |     |     |                | Year the song was released.                           |
| explicit         | BOOLEAN      |    No    |     |     | FALSE          | Indicates whether the song contains explicit content. |

---

## Business Rules

* Every song must have an associated artist.
* Album association is optional because some songs are released as standalone singles.
* Song duration must be greater than zero.
* Release year cannot exceed the current year.
* Genre values must be standardized during ETL.

---

| Rule                                   | Expected Result |
| -------------------------------------- | --------------- |
