# 6.13 Dimension Table — `dim_episode`

## Business Purpose

The `dim_episode` table stores descriptive information about individual podcast episodes.

It allows analysts to evaluate podcast performance at the episode level and connect listening behavior to podcast shows.

---

## Grain

**One row represents one podcast episode.**

Example:

| episode_key | episode_title                         |
| ----------- | ------------------------------------- |
| 1           | Episode 001: The Future of AI         |
| 2           | Episode 002: Climate Change Explained |

---

## Source

Derived from the operational `Podcast_Episodes` table during ETL.

---

## Primary Key

`episode_key`

---

## Referenced By

* fact_podcast_play

---

## References

* `podcast_key` → dim_podcast

---

## Column Definitions

| Column           | Data Type    | Nullable |  PK |  FK | Default        | Description                                      |
| ---------------- | ------------ | :------: | :-: | :-: | -------------- | ------------------------------------------------ |
| episode_key      | INTEGER      |    No    |  ✓  |     | Auto-generated | Surrogate key.                                   |
| episode_id       | VARCHAR(20)  |    No    |     |     |                | Business identifier from the operational system. |
| podcast_key      | INTEGER      |    No    |     |  ✓  |                | Foreign key referencing the podcast dimension.   |
| episode_title    | VARCHAR(255) |    No    |     |     |                | Episode title.                                   |
| duration_seconds | INTEGER      |    No    |     |     |                | Episode duration.                                |
| release_date     | DATE         |    No    |     |     |                | Episode publication date.                        |

---

## Business Rules

* Every episode belongs to exactly one podcast.
* Episode duration must be greater than zero.
* Release date cannot occur in the future.
* Episode IDs must be unique.

---

## Data Quality Rules

| Rule                           | Expected Result |
| ------------------------------ | --------------- |
| No duplicate episode_id values | Pass            |
| Podcast relationship exists    | Pass            |
| Duration is valid              | Pass            |
| Release date is valid          | Pass            |

---

## Estimated Size

Approximately **15,000 rows**.
