# 6.12 Dimension Table — `dim_podcast`

## Business Purpose

The `dim_podcast` table stores descriptive information about podcast shows available on the Pulse Music platform.

It supports podcast-level analysis including audience growth, category performance, and publisher trends.

---

## Grain

**One row represents one podcast show.**

Example:

| podcast_key | podcast_name      |
| ----------- | ----------------- |
| 1           | The Daily Byte    |
| 2           | Science Explained |
| 3           | Business Weekly   |

---

## Source

Derived from the operational `Podcasts` table during ETL.

---

## Primary Key

`podcast_key`

---

## Referenced By

* dim_episode

---

## Column Definitions

| Column       | Data Type    | Nullable |  PK |  FK | Default        | Description                                      |
| ------------ | ------------ | :------: | :-: | :-: | -------------- | ------------------------------------------------ |
| podcast_key  | INTEGER      |    No    |  ✓  |     | Auto-generated | Surrogate key.                                   |
| podcast_id   | VARCHAR(20)  |    No    |     |     |                | Business identifier from the operational system. |
| podcast_name | VARCHAR(255) |    No    |     |     |                | Podcast title.                                   |
| category     | VARCHAR(100) |    No    |     |     |                | Primary podcast category.                        |
| publisher    | VARCHAR(255) |    Yes   |     |     |                | Organization or individual publisher.            |

---

## Business Rules

* Each podcast must have a unique podcast_id.
* Category values must be standardized.
* A podcast may contain many episodes.
* A podcast cannot exist without a unique identifier.

---

## Data Quality Rules

| Rule                           | Expected Result |
| ------------------------------ | --------------- |
| No duplicate podcast_id values | Pass            |
| Podcast name cannot be NULL    | Pass            |
| Category cannot be NULL        | Pass            |

---

## Estimated Size

Approximately **1,000 rows**.
