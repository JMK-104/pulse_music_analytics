# 6.6 Dimension Table — `dim_payment_method`

## Business Purpose

The `dim_payment_method` table standardizes the payment methods used for subscription transactions. It supports financial reporting, payment success analysis, refund analysis, and customer payment preferences.

This dimension is intentionally small and highly stable.

---

## Grain

**One row represents one payment method.**

Example:

| payment_method_key | payment_method |
| ------------------ | -------------- |
| 1                  | Credit Card    |
| 2                  | PayPal         |
| 3                  | Apple Pay      |

---

## Source

Derived from the operational `Payments` table during ETL and standardized into approved values.

---

## Primary Key

`payment_method_key`

---

## Referenced By

* fact_payment

---

## Column Definitions

| Column             | Data Type   | Nullable |  PK |  FK | Default        | Description                       |
| ------------------ | ----------- | :------: | :-: | :-: | -------------- | --------------------------------- |
| payment_method_key | INTEGER     |    No    |  ✓  |     | Auto-generated | Surrogate key.                    |
| payment_method     | VARCHAR(50) |    No    |     |     |                | Standardized payment method name. |

---

## Business Rules

Supported payment methods include:

* Credit Card
* PayPal
* Apple Pay
* Google Pay
* Gift Card

Operational values are standardized during ETL to ensure consistency.

For example:

* VISA → Credit Card
* Mastercard → Credit Card
* AMEX → Credit Card

---
