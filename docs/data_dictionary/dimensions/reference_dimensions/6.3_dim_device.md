# 6.3 Dimension Table — `dim_device`

## Business Purpose

The `dim_device` table standardizes the devices used to access the Pulse Music platform. It supports reporting on listening behavior, session duration, and podcast consumption across device types and operating systems.

---

## Grain

**One row represents one unique device and operating system combination.**

Example:

| device_key | device_type | operating_system |
| ---------- | ----------- | ---------------- |
| 1          | Smartphone  | Android          |
| 2          | Smartphone  | iOS              |
| 3          | Desktop     | Windows          |

---

## Source

Derived from the operational Sessions and Users tables during ETL.

---

## Primary Key

`device_key`

---

## Referenced By

* fact_listening
* fact_session
* fact_podcast_play

---

## Column Definitions

| Column           | Data Type   | Nullable |  PK |  FK | Default        | Description                                                             |
| ---------------- | ----------- | :------: | :-: | :-: | -------------- | ----------------------------------------------------------------------- |
| device_key       | INTEGER     |    No    |  ✓  |     | Auto-generated | Surrogate key.                                                          |
| device_type      | VARCHAR(30) |    No    |     |     |                | Device category (Smartphone, Desktop, Tablet, Smart Speaker, TV, etc.). |
| operating_system | VARCHAR(30) |    No    |     |     |                | Operating system running on the device.                                 |

---

## Business Rules

* Each unique combination of `device_type` and `operating_system` appears only once.
* Devices are standardized during ETL to prevent duplicate values (e.g., "Android", "android", and "ANDROID" all become "Android").

---

## Data Quality Rules

| Rule                             | Expected Result |
| -------------------------------- | --------------- |
| No duplicate device combinations | Pass            |
| Device type not null             | Pass            |
| Operating system not null        | Pass            |

---

## Estimated Size

Approximately **10–20 rows**, depending on the number of supported device/OS combinations.
