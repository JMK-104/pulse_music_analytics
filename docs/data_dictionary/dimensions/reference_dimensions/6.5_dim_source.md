# 6.5 Dimension Table — `dim_source`

## Business Purpose

The `dim_source` table identifies how playback was initiated. It enables analysts to understand user discovery behavior by distinguishing whether a song or podcast was played from search, playlists, recommendations, albums, radio, or the user's library.

This dimension supports engagement and recommendation analyses.

---

## Grain

**One row represents one playback source.**

Example:

| source_key | source_name    |
| ---------- | -------------- |
| 1          | Search         |
| 2          | Playlist       |
| 3          | Recommendation |

---

## Source

Reference data generated during warehouse initialization.

This table is not loaded directly from the operational database.

---

## Primary Key

`source_key`

---

| source_key  | INTEGER     |    No    |  ✓  |     | Auto-generated | Surrogate key.                        |
| source_name | VARCHAR(50) |    No    |     |     |                | Describes how playback was initiated. |

---

## Business Rules

The initial warehouse implementation supports the following playback sources:

* Search
* Playlist
* Recommendation
* Album
* Radio
* Library

Additional playback sources may be added as the platform evolves.
