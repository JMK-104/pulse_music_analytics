# 6.4 Dimension Table — `dim_plan`

## Business Purpose

The `dim_plan` table stores the subscription plans offered by Pulse Music. It provides descriptive information about each plan, allowing analysts to evaluate user behavior, subscription trends, revenue, and engagement across different plan types.

This is a slowly changing reference dimension that contains a small number of records and is shared across multiple fact tables.

---

## Grain

**One row represents one subscription plan.**

Example:

| plan_key | plan_name          |
| -------- | ------------------ |
| 1        | Free               |
| 2        | Premium Individual |
| 3        | Premium Family     |
| 4        | Student            |

---

## Source

Derived from the operational `Subscription_Plans` table during ETL.

---

## Primary Key

`plan_key`

---

## Referenced By

* fact_listening
* fact_subscription_event
* fact_payment
* fact_podcast_play

---

## Column Definitions

| Column           | Data Type     | Nullable |  PK |  FK | Default        | Description                                             |
| ---------------- | ------------- | :------: | :-: | :-: | -------------- | ------------------------------------------------------- |
| plan_key         | INTEGER       |    No    |  ✓  |     | Auto-generated | Surrogate key used throughout the warehouse.            |
| plan_id          | VARCHAR(20)   |    No    |     |     |                | Business identifier from the operational system.        |
| plan_name        | VARCHAR(50)   |    No    |     |     |                | Subscription plan name.                                 |
| monthly_price    | NUMERIC(10,2) |    No    |     |     |                | Standard monthly subscription price.                    |
| ad_free          | BOOLEAN       |    No    |     |     | FALSE          | Indicates whether advertisements are removed.           |
| offline_download | BOOLEAN       |    No    |     |     | FALSE          | Indicates whether offline listening is supported.       |
| max_accounts     | SMALLINT      |    No    |     |     | 1              | Maximum number of user profiles allowed under the plan. |

---

## Business Rules

* Each subscription plan appears only once.
* Plan names must be unique.
* Monthly price cannot be negative.
* The Free plan must always have a monthly price of 0.00.
* Premium plans must have `ad_free = TRUE`.
* Offline downloads are available only on Premium plans.

---

## Data Quality Rules

| Rule                               | Expected Result |
| ---------------------------------- | --------------- |
| No duplicate plan_id values        | Pass            |
| No duplicate plan_name values      | Pass            |
| Monthly price ≥ 0                  | Pass            |
| Premium plans have ad_free = TRUE  | Pass            |
| Free plan has monthly_price = 0.00 | Pass            |
